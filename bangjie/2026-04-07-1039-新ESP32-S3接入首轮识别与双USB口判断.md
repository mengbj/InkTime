# 新ESP32-S3接入首轮识别与双USB口判断

记录时间：2026-04-07 10:39（Asia/Shanghai）

## 本轮目标

用户已更换新的 `ESP32-S3` 开发板，希望重新推进：
- 先确认 Mac 能识别该板；
- 再完成烧录；
- 烧录确认无误后，再继续接入 7.3 寸四色墨水屏。

## 从图片识别到的关键信息

从用户提供的板卡照片可见：
- 模组为 `ESP32-S3-WROOM`
- 模组丝印可见 `N16R8` 信息，推断这块板大概率是：
  - `16MB Flash`
  - `8MB PSRAM`
- 板上有两个 `USB-C` 接口
- 板上有：
  - `RST`
  - `BOOT`
  - `RGB`
- 底部右侧有一颗独立 USB/串口桥芯片，说明这块板很可能同时具备：
  - 一个 `USB-UART` 口
  - 一个 `Native USB / OTG` 口

这类双 USB-C 的 `ESP32-S3` 板子，最容易出现的第一类问题就是：
- **插到了能供电但不走默认烧录链路的那个口**。

## 本机当前实际检测结果

我刚刚在这台 Mac 上检查了串口设备：
- 当前只有：
  - `/dev/cu.Bluetooth-Incoming-Port`
  - `/dev/cu.debug-console`

也就是说：
- **系统当前还没有识别到新的 ESP32-S3 串口设备**。

这一步非常关键，因为只要串口还没出现，就还不能进入 Arduino IDE 正常烧录阶段。

## 当前结论

目前并不是“烧录失败”，而是更前一步：
- **Mac 还没识别到这块新 S3 板子的可用串口**。

结合板型特征，当前优先怀疑：
1. 插在了不对的 USB-C 口；
2. 线插得通电但没有建立数据链路；
3. 板子没有通过当前这个口暴露出默认串口。

## 这块板子当前最应该怎么接

先不要接墨水屏。

先只保留：
- `ESP32-S3 裸板`
- `一根已验证可传数据的 USB 线`
- `Mac`

并优先尝试：
- **把 USB 线插到更靠近右侧那颗 USB/串口桥芯片的那个 USB-C 口**。

原因：
- 这颗桥接芯片通常对应默认的串口烧录口；
- Arduino IDE / esptool 走这个口成功率最高。

## 现在立刻执行的步骤

### 第一步：只接开发板，不接屏
- 墨水屏和转接板全部先不要接
- 杜邦线也都不要接
- 只保留开发板和 USB

### 第二步：换到另一个 USB-C 口
如果你现在插的是左边口，就换右边口；
如果现在插的是右边口，就换左边口。

但优先建议先试：
- **靠近板上 USB/串口桥芯片的那个口**。

### 第三步：重新观察 Mac 是否出现串口
换口后，目标是让 Arduino IDE 或系统里出现类似：
- `/dev/cu.usbmodem...`
- 或 `/dev/cu.wchusbserial...`
- 或 `/dev/cu.SLAB_USBtoUART`

只要出现新的 `cu.*` 串口，我们就进入下一步烧录。

## 一旦串口出现，Arduino IDE 设置就按这个来

- `Board = ESP32S3 Dev Module`
- `Flash Size = 16MB`
- `PSRAM = OPI PSRAM`
- `USB CDC On Boot = Enabled`
- `Upload Speed = 115200`

然后先烧一个最小串口测试程序，确认：
- 能写入；
- 能复位；
- 串口监视器能打印日志；
- 再切回 `ink-display-7C-photo.ino`。
