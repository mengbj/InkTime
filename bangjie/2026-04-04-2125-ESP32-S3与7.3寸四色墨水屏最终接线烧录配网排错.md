## 2. 对应你这块板子的最终接线

你手上的板子是通用 `ESP32-S3 XH-S3E / N16R8`，墨水屏这边是 8Pin 转接板，丝印为：

- `BUSY`
- `RST`
- `DC`
- `CS`
- `SCL`
- `SDA`
- `GND`
- `3V3`

这里一定要注意：

- 这块转接板上的 `SCL / SDA` 是 **SPI 命名**，不是 I2C
- 也就是：
  - `SCL = SPI SCLK`
  - `SDA = SPI MOSI / DIN`

### 最终接线表

| 墨水屏转接板 | ESP32-S3 开发板 |
| --- | --- |
| BUSY | `GPIO14` |
| RST | `GPIO13` |
| DC | `GPIO12` |
| CS | `GPIO11` |
| SCL | `GPIO10` |
| SDA | `GPIO9` |
| GND | `GND` |
| 3V3 | `3V3` |

### 两个必须遵守的原则

1. **只能接 3V3，不能接 5V。**
2. **不要把这组 `SDA/SCL` 当 I2C 接。** 这套方案里它们是 SPI。

## 6. 这次遇到的实际错误，以及我怎么处理的

在 `ESP32 Arduino Core 3.3.7` 下，原始代码第一次编译时报错：

```text
error: 'ESP_PD_DOMAIN_RTC_SLOW_MEM' was not declared in this scope
error: 'ESP_PD_DOMAIN_RTC_FAST_MEM' was not declared in this scope
```

报错位置在：

- `/Users/apple/bangjie/dev_jy/InkTime/esp32/ink-display-7C-photo/ink-display-7C-photo.ino`

问题点是 `prepareDeepSleepDomains()` 里原先直接写了：

```cpp
esp_sleep_pd_config(ESP_PD_DOMAIN_RTC_PERIPH,   ESP_PD_OPTION_OFF);
esp_sleep_pd_config(ESP_PD_DOMAIN_RTC_SLOW_MEM, ESP_PD_OPTION_OFF);
esp_sleep_pd_config(ESP_PD_DOMAIN_RTC_FAST_MEM, ESP_PD_OPTION_OFF);
```

### 原因判断

我继续查了 Espressif 这边的定义，结论是：

- 在新的 `ESP32 Core 3.x / ESP-IDF 5.x` 体系下，`esp_sleep.h` 里的这些电源域枚举会受 `SOC_PM_SUPPORT_*` 宏控制
- 对 `ESP32-S3` 来说，`RTC_PERIPH` 还支持
- 但 `RTC_SLOW_MEM` / `RTC_FAST_MEM` 在当前 S3 目标下并不会导出，所以直接写死会编译失败

我本地查到的官方相关资料：

- Espressif `ESP32-S3 Sleep Modes` 文档：[Sleep Modes](https://docs.espressif.com/projects/esp-idf/en/stable/esp32s3/api-reference/system/sleep_modes.html)
- Espressif `esp_sleep.h` 源码（枚举受 `SOC_PM_SUPPORT_RTC_SLOW_MEM_PD` / `SOC_PM_SUPPORT_RTC_FAST_MEM_PD` 控制）：[esp_sleep.h](https://github.com/espressif/esp-idf/blob/master/components/esp_hw_support/include/esp_sleep.h)
- Espressif `esp32s3 soc_caps.h`（我本地核对到 S3 只定义了 `SOC_PM_SUPPORT_RTC_PERIPH_PD`，未定义 slow/fast mem 对应支持宏）：[soc_caps.h](https://github.com/espressif/esp-idf/blob/master/components/soc/esp32s3/include/soc/soc_caps.h)

### 处理方式

我已经把代码改成了“按目标芯片能力编译”的写法：

```cpp
void prepareDeepSleepDomains() {
  esp_sleep_pd_config(ESP_PD_DOMAIN_RTC_PERIPH, ESP_PD_OPTION_OFF);
#if defined(SOC_PM_SUPPORT_RTC_SLOW_MEM_PD) && SOC_PM_SUPPORT_RTC_SLOW_MEM_PD
  esp_sleep_pd_config(ESP_PD_DOMAIN_RTC_SLOW_MEM, ESP_PD_OPTION_OFF);
#endif
#if defined(SOC_PM_SUPPORT_RTC_FAST_MEM_PD) && SOC_PM_SUPPORT_RTC_FAST_MEM_PD
  esp_sleep_pd_config(ESP_PD_DOMAIN_RTC_FAST_MEM, ESP_PD_OPTION_OFF);
#endif
}
```

这样做的好处是：

- `ESP32-S3 + Core 3.x` 能通过
- 旧目标如果确实支持 slow/fast mem，对应代码也还能保留

## 8. 首次启动与首次配网

首次烧录完成后，设备如果还没有保存 WiFi，会自动进入 AP 配网模式。

### 8.1 连接热点

连接热点：

- `InkTime-xxxx`

### 8.2 打开配置页

浏览器访问：

- [http://192.168.4.1/](http://192.168.4.1/)

### 8.3 填入信息

需要填写：

- 家里或办公室的 WiFi 名称
- WiFi 密码
- 服务器地址，例如：`192.168.1.20:8765`

### 8.4 特别注意

**不要填 `127.0.0.1:8765`。**

原因很简单：

- 对 ESP32 来说，`127.0.0.1` 指向的是它自己
- 不是你电脑上跑的服务端

所以这里一定要填：

- 你电脑在局域网中的真实 IP
- 加上端口

例如：

- `192.168.1.20:8765`

保存后，设备会重启，然后尝试：

1. 连 WiFi
2. 请求服务器
3. 下载当天图片
4. 刷新到墨水屏

## 10. 串口与烧录排错最终版

这是你后面最可能碰到的部分，我按实际概率排序。

### 10.1 电脑根本看不到新串口

如果 `Tools -> Port` 里没有新设备，先看下面几项：

1. **USB 线是不是纯充电线**
   - 很多 Type-C 线只能供电，不能传数据
   - 换一根确定能传数据的线，优先排这个

2. **USB 口是不是松动 / 接触不良**
   - 重新插拔
   - 换电脑另一侧接口
   - 尽量别先经过廉价扩展坞

3. **板子有没有真的上电**
   - 看板载电源灯是否正常

4. **macOS 当前看到的设备是不是只有系统自带串口**
   - 如果只看到：
     - `/dev/cu.Bluetooth-Incoming-Port`
     - `/dev/cu.debug-console`
   - 那基本说明 ESP32 板子没有被系统识别出来

### 10.2 点击 Upload 后一直 `Connecting...`

先试最常见的两招：

1. 把 `Upload Speed` 从 `921600` 改到 `460800`
2. 按住 `Boot`，再点上传；出现 `Connecting...` 后松开 `Boot`

如果还不行，再试：

- 点上传时按一下 `RST`
- 或者先按住 `Boot`，再短按 `RST`，等进入下载模式后松开

### 10.3 烧录能过，但串口日志乱码或没有输出

先确认：

- 串口监视器波特率是 `115200`
- `USB CDC On Boot` 已启用

这个工程当前串口调试宏是开启的：

```cpp
#define DEBUG_LOG 1
```

所以正常情况下，上电后应有调试输出。

### 10.4 烧录成功，但没有出现 `InkTime-xxxx`

重点排这几项：

1. 板子是否真的完成启动，而不是不停重启
2. 是否因为供电不稳导致 WiFi 启动时复位
3. 是否接线短路，尤其是 `3V3 / GND`
4. 是否墨水屏转接板把主板电源拖垮

建议临时把串口打开，先看日志里有没有：

- WiFi 初始化失败
- 内存分配失败
- 启动后立刻重启

### 10.5 热点能连，`192.168.4.1` 打不开

一般是下面几种情况：

1. 手机或电脑没有真正连上 `InkTime-xxxx`
2. 系统还在自动走蜂窝或原 WiFi
3. 浏览器缓存了旧页面
4. 设备其实已经重启退出了 AP 模式

排查顺序：

- 先确认当前网络就是 `InkTime-xxxx`
- 再用浏览器访问 [http://192.168.4.1/](http://192.168.4.1/)
- 不行就换无痕模式或换另一台设备

### 10.6 WiFi 配好了，但一直拉不到图片

优先检查三件事：

1. 填的是不是 `127.0.0.1:8765`
   - 如果是，改成电脑局域网 IP

2. 电脑和 ESP32 是否在同一个局域网
   - 比如都在同一个路由器的 `192.168.1.x`

3. `DOWNLOAD_KEY` 是否一致
   - 不一致很容易导致 `404`

### 10.7 编译时报内存或 framebuffer 相关问题

重点检查：

1. `PSRAM` 是否真的打开
2. 是否选成了错误的板卡
3. 是否误用了没有 PSRAM 的板型配置

这块板子是 `N16R8`，优先就按：

- `Flash Size = 16MB`
- `PSRAM = OPI PSRAM`

### 10.8 屏幕完全不刷新，但串口看起来正常

这时优先看硬件：

1. `BUSY / RST / DC / CS / SCL / SDA` 是否一一对应
2. `3V3` 有没有错接到 `5V`
3. `SCL / SDA` 是否误接成 I2C 认知
4. 杜邦线是否接触不良
5. 屏幕排线和转接板是否压紧

### 10.9 想清空配网重新来

当前固件里恢复出厂设置脚是：

- `GPIO38`

逻辑是：

- 上电时若 `GPIO38` 被拉低，就清空已保存的 WiFi 与服务器配置

所以如果后面你要做“恢复出厂”按钮，可以：

- `GPIO38` 接按键
- 按键另一端接 `GND`

上电按住即可清配置。
