## 4. 下一步最优动作

现在不要继续盲目重复同样上传动作，下一步请按下面顺序做。

### 第一步：物理上再次确认“裸板状态”

请确认当前是否真的已经：

- 拔掉了墨水屏转接板到 ESP32 的所有杜邦线
- 只保留 ESP32-S3 + USB 线

如果**还连着屏幕或转接板**，那先全部拔掉，再试。

### 第二步：手动进入下载模式再上传

请按住开发板上的按键，严格按这个顺序：

1. 按住 `Boot`
2. 短按一下 `RST`
3. 松开 `RST`
4. 继续按住 `Boot` 约 2 秒
5. 松开 `Boot`
6. 立刻点击 Arduino IDE 左上角 `Upload`

### 第三步：如果还是同样报错，再做链路隔离验证

如果手动下载模式后还是 `0107`，那下一步就要优先怀疑：

- USB 线仍然不稳定
- USB 口或扩展坞链路不稳
- 这块板子的 USB 串口桥 / 板载下载链路本身不稳定

那时建议继续按顺序排：

1. 换另一根确定能传数据的 USB 线
2. 不经过扩展坞，直接插 Mac 本体接口
3. 换一个 USB 口再试
4. 再次保持裸板状态上传
