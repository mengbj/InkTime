# 裸板手动下载模式仍报 0107 的结论与下一步

## 1. 本轮实际结果

本轮已经在以下条件下再次验证：

- 墨水屏屏线已全部拔掉
- 只保留 `ESP32-S3` 裸板
- 已按 `Boot -> RST -> 松RST -> 等2秒 -> 松Boot -> Upload` 手动进入下载模式

但 Arduino IDE 输出仍然是：

```text
Failed to write to target RAM (result was 0107: Checksum error)
Uploading stub flasher...
Failed uploading: uploading error: exit status 2
```

这说明：

- **问题已经基本排除“墨水屏外设干扰”这一项**
- 当前阻塞更集中在：
  - USB 线质量
  - USB 口 / 转接链路
  - 板载 USB 串口桥
  - 板子本体下载写入稳定性

---

## 2. 截至当前已经排除掉什么

现在已经排除或基本排除：

- 代码编译问题
- Arduino IDE 参数设置问题
- 板卡选择错误
- 端口选择错误
- 墨水屏当前接线干扰
- 不进下载模式这一类基础操作问题

因为本轮已经证明：

1. 固件能编译通过
2. 芯片能被识别为 `ESP32-S3`
3. 裸板状态下仍然复现同样 `0107`

---

## 3. 当前最可能的原因排序

根据前面几轮实测，当前最可能原因优先级大致是：

### 第一优先级

- **USB 数据线仍然不稳定**

即使能识别串口，也可能在大包写 RAM / 写 Flash 时出错。

### 第二优先级

- **当前 USB 口或中间链路不稳定**

例如：

- 转接头
- 扩展坞
- 某个 Type-C 口接触边缘不稳

### 第三优先级

- **这块 ESP32-S3 板子的下载链路或串口桥本身有问题**

也就是：

- 能识别芯片
- 但一到真正写入阶段就不稳定

---

## 4. 现在的最优下一步

下一轮不要再先改代码，也不要先接回屏幕。

现在最优动作是：

### 方案 A：换 USB 线再试

请换一根你确定稳定、能传数据、最好以前给开发板烧录成功过的线。

### 方案 B：换 Mac 上另一个 USB 口

要求：

- 尽量不要经过扩展坞
- 直接插电脑本体接口

### 方案 C：换线后继续保持裸板状态再上传

仍然保持：

- 无屏线
- 无转接板连线
- 只留 ESP32-S3 + USB

然后再试一次 Arduino IDE 上传。

---

## 5. 如果下一轮还失败，结论会更明确

如果换线、换 USB 口、继续裸板后，还是同样 `0107`，那就很接近以下判断：

- **这块板子本体下载写入链路有硬件稳定性问题**

那时再继续折腾屏幕已经没有意义，应该优先：

1. 更换另一块 `ESP32-S3` 板子
2. 或改走外部 USB-UART 下载方案
