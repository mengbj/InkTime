# RST后WiFi连接失败回退AP模式排查

记录时间：2026-04-07 14:02（Asia/Shanghai）

## 本轮现象

用户在按下 ESP32-S3 的 `RST` 之后，Arduino IDE 串口监视器显示：

- `[CFG] valid=true`
- `[BOOT] have config -> connect WiFi`
- `[WIFI] connectWiFi()`
- `[WIFI] target ssid=mengbj_iPhone`
- `[WIFI] connect FAILED`
- `[BOOT] connect failed -> AP portal`
- `[CFG] enter startConfigPortal()`
- `[CFG] AP SSID = InkTime-090`
- `[CFG] AP IP = 192.168.4.1`

这说明：

- 之前保存的配置还在，配置没有丢
- 这次失败点不是 HTTP 下载，也不是墨水屏驱动
- 当前失败发生在 **ESP32 连接手机热点这一步**
- 固件因此自动回退到了配网 AP 模式 `InkTime-090`

3. ESP32 当前启动路径是：
   - 已读取到保存配置

   - 自动拉起 `InkTime-090`

**现在要先解决“ESP32 为什么没连上手机热点”，然后才会继续进入下载 `.bin` 和点亮墨水屏的步骤。**

因为流程还停在更前面：

- **Wi-Fi 连接阶段**

## 最可能的原因

结合当前现象，优先怀疑以下几种情况：

### 1）手机热点密码填错 / 被改过

ESP32 日志只显示 `connect FAILED`，最常见原因之一就是：

- 手机热点密码和之前输入到 ESP32 的密码不一致
- 或者用户手动改过热点密码

### 2）手机热点刚才短暂不可用

例如：

- iPhone 锁屏后一段时间热点行为变化
- 热点列表中显示存在，但实际没有稳定接受新设备接入
- 手机热点临时抖动，ESP32 这次连接超时

### 3）ESP32 之前保存的是旧密码

即便 SSID 还是 `mengbj_iPhone`，如果之前配网时输错过一次密码，而配置被保存了，也会一直失败，直到重新配网覆盖。

### 4）手机热点频段 / 兼容性抖动

ESP32 只能连 2.4GHz Wi-Fi。虽然 iPhone 热点通常能兼容，但偶发也会出现重新广播、兼容模式切换、首次失败后重试成功等情况。

## 现在最稳的继续步骤

### 第 1 步：立刻连接到 ESP32 自己的热点

因为它已经回退到了 AP 模式，所以现在先用手机或 Mac 连接：

- Wi-Fi 名称：`InkTime-090`
- 密码：`12345678`

连接后打开：

- `http://192.168.4.1/`

---

### 第 2 步：重新填写热点参数

在配网页里重新输入：

- WiFi SSID：`mengbj_iPhone`
- WiFi Password：**重新手动输入一遍你现在手机热点的密码**
- Server：`172.20.10.9:8765`

这里最重要的是：

- 不要直接默认之前是对的
- 这次重新手敲一遍热点密码

---

### 第 3 步：保存后观察串口

保存后，看串口是否变成下面这种路径：

- `[WIFI] target ssid=mengbj_iPhone`
- `[WIFI] connected`
- `[WIFI] IP=172.20.10.x`
- `[TIME] OK: ...`
- `HTTP code=200`

如果能到这里，说明 Wi-Fi 问题已经解除，后面就开始进入下载和刷屏。

---

## 如果你不想重新配网，也可以先做一个快检查

在 iPhone 热点页面里确认两件事：

- 热点确实还是开启状态
- 热点密码没有被改过

但从效率上讲，**现在直接重新进 `InkTime-090` 配网页重填一遍密码，是最快的办法。**
