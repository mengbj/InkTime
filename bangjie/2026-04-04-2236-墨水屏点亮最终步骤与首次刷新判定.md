# ESP32-S3 与 7.3 寸四色墨水屏点亮最终步骤与首次刷新判定

## 1. 先说结论：这块屏怎么才算真正“点亮”

结合当前仓库固件、你已经接好的实物照片，以及我前面实际推进到的烧录状态，这块 7.3 寸四色墨水屏要真正点亮，顺序不是“上电就亮”，而是：

1. `ESP32-S3` 先成功烧录 `esp32/ink-display-7C-photo/ink-display-7C-photo.ino`
2. 首次启动后，设备进入配网热点 `InkTime-xxxx`
3. 手机或电脑连上热点，打开 `http://192.168.4.1/`
4. 填入家里 WiFi 和服务器地址，例如 `192.168.1.20:8765`
5. ESP32 重启后连上局域网
6. 它去服务器下载当天的四色 BIN 图
7. `GxEPD2_730c_GDEY073D46` 驱动执行一次完整刷新
8. 屏幕出现新内容，这才算当前链路真正点亮

也就是说，**你现在屏幕上已经残留的日期/文字，不代表当前 ESP32 已经成功驱动它**。墨水屏断电后本来就会保留上一次画面，所以“屏上有内容”只能说明它以前被刷过，不能说明这次接线和当前程序已经跑通。

- **先裸板烧录成功**
- **再把屏重新接回去启动**

## 3. 当前这块板子的确认接线

当前项目 `esp32/ink-display-7C-photo/ink-display-7C-photo.ino` 里定义的引脚就是：

```cpp
#define PIN_EPD_BUSY 14
#define PIN_EPD_RST  13
#define PIN_EPD_DC   12
#define PIN_EPD_CS   11
#define PIN_EPD_SCLK 10
#define PIN_EPD_DIN  9
```

对应转接板丝印，应接成：

| 墨水屏转接板 | ESP32-S3 |
| --- | --- |
| `BUSY` | `GPIO14` |
| `RST` | `GPIO13` |
| `DC` | `GPIO12` |
| `CS` | `GPIO11` |
| `SCL` | `GPIO10` |
| `SDA` | `GPIO9` |
| `GND` | `GND` |
| `3V3` | `3V3` |

两个必须记住的点：

1. `SCL/SDA` 在这块转接板上是 **SPI**，不是 I2C
2. 供电必须是 `3V3`，**不能接 `5V`**

## 4. 真正点亮的最稳妥操作顺序

### 第一步：先把屏和转接板临时拔掉，只留 ESP32-S3 裸板

原因不是接线错，而是前面真实烧录日志已经证明，当前问题卡在：

- `Uploading stub flasher`
- `Failed to write to target RAM (0107: Checksum error)`

以及：

- `Failed to write to target flash after seq 0 (0105: The format of the received message is invalid)`

这类错误非常像：

- 外设干扰下载期引脚状态
- 屏或转接板拖了供电
- SPI/控制线让烧录链路不稳定

所以最优顺序是：

1. 拔掉墨水屏转接板到 ESP32 的所有线
2. 只保留 ESP32-S3 和 Mac 的 USB 连接
3. 先完成 Upload
4. 成功后断电
5. 再按接线表把屏重新接回去

### 第二步：在 Arduino IDE 里按下面参数烧录

建议参数：

- `Board` -> `ESP32S3 Dev Module`
- `Flash Size` -> `16MB`
- `PSRAM` -> `OPI PSRAM`
- `USB CDC On Boot` -> `Enabled`
- `Upload Speed` -> 先用 `115200` 或 `460800`

打开文件：

- `/Users/apple/bangjie/dev_jy/InkTime/esp32/ink-display-7C-photo/ink-display-7C-photo.ino`

如果点 `Upload` 还是卡连接，手动进下载模式：

1. 按住 `Boot`
2. 短按 `RST`
3. 松开 `RST`
4. 继续按住 `Boot` 1 到 2 秒
5. 松开 `Boot`
6. 立刻再点上传

### 第三步：烧录成功后，再把屏接回去

重新接回：

- `GPIO9 ~ GPIO14`
- `3V3`
- `GND`
- 屏幕排线确认压紧在绿色转接板上

然后给板子重新上电。

## 5. 这份固件上电后，屏幕会发生什么

这一点非常关键。

这份固件的启动逻辑不是“上电立刻画一个本地测试页”，而是：

### 情况 A：还没有保存 WiFi / 服务器配置

它会进入配网模式：

- 开热点：`InkTime-xxxx`
- 配网页：`http://192.168.4.1/`

这时**不一定马上刷屏**。因为当前程序的主要显示动作，是在：

- 连上 WiFi
- 同步时间
- 下载当天 BIN 图片

之后才调用：

- `initDisplay(g_cfg)`
- `drawFromFramebuffer(g_cfg)`

### 情况 B：已经保存过配置，且网络正常

它会：

1. 连 WiFi
2. 请求你的服务端
3. 下载当天图片 BIN
4. 全刷一次电子纸
5. 进入深度睡眠，等待下一次刷新时间

所以如果你现在想看到“确实被当前程序驱动”的结果，最重要的不是只盯着屏，而是同时确认：

- 热点是否出来
- 配网页是否能打开
- 串口日志是否跑到 `WIFI / HTTP / EPD` 阶段

## 6. 首次配网怎么做，做完后屏才会刷新

### 6.1 连热点

首次启动后，用手机或电脑连接：

- `InkTime-xxxx`

### 6.2 打开配置页

浏览器访问：

- `http://192.168.4.1/`

### 6.3 填写配置

至少要填：

- 家里的 `WiFi SSID`
- `WiFi Password`
- `服务器地址`

服务器地址要写局域网 IP，例如：

- `192.168.1.20:8765`

不要写：

- `127.0.0.1:8765`

因为那对 ESP32 来说指向的是它自己，不是你电脑。

### 6.4 保存后会发生什么

保存后设备重启，然后会尝试：

1. 连入你填的 WiFi
2. 访问服务端
3. 下载 `/static/inktime/yourdownloadkey/photo_x.bin`
4. 刷屏

如果这一步成功，你就会看到旧残影被新内容替换掉。

## 7. 怎么判断“真的点亮成功”

下面这些现象，才说明链路真的通了。

### 成功标志 1：能看到 `InkTime-xxxx`

这说明：

- 程序至少已经启动
- WiFi AP 模式跑起来了

### 成功标志 2：`http://192.168.4.1/` 能打开

这说明：

- `WebServer` 跑起来了
- 配网页逻辑正常

### 成功标志 3：保存 WiFi 后设备不再回到 AP，而是正常联网

这说明：

- 配置已写入 NVS
- ESP32 能正常进 STA 模式

### 成功标志 4：串口日志出现这些关键字

建议打开 Arduino 串口监视器，波特率 `115200`，重点看：

- `[BOOT] have config -> connect WiFi`
- `[WIFI] connected`
- `[HTTP] GET http://...`
- `[EPD] initDisplay`

如果能看到 `EPD` 相关日志，说明程序已经走到真正驱动墨水屏的步骤。

### 成功标志 5：屏幕出现一次完整刷新

电子纸正常全刷时，通常会出现：

- 闪屏或阶段性刷新
- 旧内容被清除
- 新内容稳定停留

这时才是真正意义上的“点亮”。

当前最短路径如下：

1. **先拔掉屏线，只烧裸板**
2. **Upload 成功后断电**
3. **再接回屏线**
4. **重新上电**

7. **看串口日志有没有走到 `[EPD] initDisplay`**
8. **等待第一次完整刷新**

这是目前最现实、最稳的点亮路径。

## 9. 如果热点出来了，但屏还是不刷新，优先查什么

### 9.1 先查服务器地址

最常见的错法就是填了：

- `127.0.0.1:8765`

正确应该填：

- 电脑在局域网里的真实地址，例如 `192.168.1.20:8765`

### 9.2 再查 `DOWNLOAD_KEY`

当前固件里写的是：

```cpp
#define DAILY_PHOTO_PATH_PREFIX "/static/inktime/yourdownloadkey/photo_"
```

所以服务端配置里的下载 key 也必须一致。否则就会表现成：

- WiFi 已连上
- HTTP 能请求
- 但图片返回 `404`
- 屏幕不刷新

### 9.3 再查接线

重点确认：

- `BUSY -> GPIO14`
- `RST -> GPIO13`
- `DC -> GPIO12`
- `CS -> GPIO11`
- `SCL -> GPIO10`
- `SDA -> GPIO9`
- `3V3 -> 3V3`
- `GND -> GND`

### 9.4 再查供电和接触

电子纸这类转接板最怕：

- 面包板虚接
- 杜邦线松动
- 排线没压紧
- `3V3` 电源接触不稳定

如果串口日志里已经出现 `[EPD] initDisplay`，但屏一点变化都没有，硬件接触问题的概率就很高。

## 10. 串口排错时最值得盯的几个日志

这份固件已经开了调试输出：

```cpp
#define DEBUG_LOG 1
```

所以串口监视器里最值得盯的是下面几段：

### 10.1 配网入口

- `[BOOT] no valid config -> AP portal`
- `[CFG] AP SSID = InkTime-xxxx`
- `[CFG] AP IP = 192.168.4.1`

说明已进入首次配网流程。

### 10.2 联网阶段

- `[WIFI] connectWiFi()`
- `[WIFI] connected`
- `[WIFI] IP=...`

说明已经连上你家的 WiFi。

### 10.3 下载阶段

- `[HTTP] GET ...`
- `[HTTP] code=200`
- `[HTTP] content-length=384000`
- `[HTTP] total read=384000`

说明当天图片 BIN 已成功下到 framebuffer。

### 10.4 刷屏阶段

- `[EPD] initDisplay`
- `[EPD] logical w=480 h=800`

说明程序已经真正开始驱动墨水屏。
