# ESP32S3选板应选哪一个

记录时间：2026-04-07 11:00（Asia/Shanghai）

## 本轮直接结论

用户截图中，当前候选项里出现了：
- `ESP32S3 Dev Module`
- `ESP32S3 Dev Module Octal (WROOM2)`

这块板子当前应优先选择：
- **`ESP32S3 Dev Module`**

而不是：
- `ESP32S3 Dev Module Octal (WROOM2)`

## 为什么选 `ESP32S3 Dev Module`

从用户板卡照片来看：
- 模组丝印是 `ESP32-S3-WROOM`
- 不是明确写着 `WROOM-2`
- 我们当前只需要一个稳定、通用、适配面最广的 `ESP32-S3` 开发板配置

因此第一轮最稳妥的做法就是：
- 先选通用项 `ESP32S3 Dev Module`

这样做的好处是：
- Arduino Core 对它支持最通用；
- 菜单里能直接看到 `PSRAM` 与 `USB CDC On Boot`；
- 便于先完成“最小程序烧录验证”；
- 便于后续继续切回 `ink-display-7C-photo.ino`。

## 为什么先不要选 `Octal (WROOM2)`

`ESP32S3 Dev Module Octal (WROOM2)` 更像是：
- 针对明确 `WROOM-2` / Octal Flash/PSRAM 组合的特定预设。

而当前从现有图片能确认的是：
- `ESP32-S3-WROOM`
- 以及很像 `N16R8`

但还没有足够证据证明：
- 就必须先用 `Octal (WROOM2)` 这个特定项。

所以按当前阶段的原则：
- **先用最通用、最稳定、最容易排障的 `ESP32S3 Dev Module`**。

如果后续发现：
- `ESP32S3 Dev Module` 下烧录/运行异常，
- 且异常与 Flash/PSRAM 模式相关，

再回头尝试 `Octal (WROOM2)` 会更合理。

## 当前这一步应该怎么点

在用户当前截图这个选择窗口里：
- 左侧板卡选：`ESP32S3 Dev Module`
- 右侧端口选：`/dev/cu.usbserial-110 Serial Port (USB)`
- 然后点：`OK`

## 选完之后，Tools 里要继续确认什么

选成 `ESP32S3 Dev Module` 后，再回到 `Tools` 菜单，继续确认：
- `PSRAM = OPI PSRAM`
- `USB CDC On Boot = Enabled`
- `Flash Size = 16MB`
- `Upload Speed = 115200`

如果 `PSRAM` 里出现：
- `Disabled`
- `QSPI PSRAM`
- `OPI PSRAM`

那就说明你已经切到正确的 `ESP32-S3` 菜单体系了。

## 本轮一句话执行指令
