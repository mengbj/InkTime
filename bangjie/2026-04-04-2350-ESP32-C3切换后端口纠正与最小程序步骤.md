# ESP32-C3切换后端口纠正与最小程序步骤

记录时间：2026-04-04 23:50（Asia/Shanghai）

## 本轮看到的实际状态

从用户截图可以明确看到：
- `Board` 已经切换为 `ESP32C3 Dev Module`
- 当前菜单显示的 `Port` 是：`/dev/cu.usbmodem101`
- `USB CDC On Boot = Enabled`
- `Upload Speed = 115200`
- `Flash Size = 4MB (32Mb)`

这说明：
- **选板已经对了**；
- **端口也已经切到新的 C3 端口**；
- 但下方黑色输出窗口里，报错仍然是：
  - `Could not open /dev/cu.usbmodem5A470123231`

## 当前结论

这说明 Arduino IDE 当前出错的这次上传，仍然是基于“上一块 S3 板子对应的旧端口”发起的，或者是：
- 旧上传任务的日志还留在窗口里；
- 当前草图还是原来的 `ink-display-7C-photo.ino`；
- IDE 还没有基于新端口 `/dev/cu.usbmodem101` 正式发起一次新的最小程序上传。

也就是说：

> 现在还不能说 C3 也烧录失败了。因为从截图看，当前失败日志对应的还是旧 S3 端口，不是新 C3 端口的真实上传结果。

## 现在走到哪一步

当前准确进度是：
- C3 板子已接入；
- `Board` 已切到 `ESP32C3 Dev Module`；
- 新端口已出现：`/dev/cu.usbmodem101`；
- 但**还没有完成一次真正针对 C3 的最小程序上传测试**。

## 下一步：不要继续用 InkTime 大工程测试

当前窗口里打开的仍是：
- `ink-display-7C-photo.ino`

这不适合作为 C3 的第一轮验证，因为：
- 它是给 `ESP32-S3 + 墨水屏` 准备的；
- 代码大、依赖多；
- 会混淆“板子烧录是否正常”和“项目本身是否兼容”这两个问题。

所以现在必须切换成 **最小测试草图**。

## 立刻照做的步骤

### 第一步：新建一个最小测试草图
在 Arduino IDE 里：
- `File -> New Sketch`

把默认内容全部删掉，替换成下面这段：

```cpp
void setup() {
  Serial.begin(115200);
  delay(1000);
  Serial.println("ESP32-C3 upload test start");
}

void loop() {
  Serial.println("tick");
  delay(1000);
}
```

### 第二步：再次确认工具菜单
确认仍然是：
- `Board = ESP32C3 Dev Module`
- `Port = /dev/cu.usbmodem101`
- `Upload Speed = 115200`
- `USB CDC On Boot = Enabled`
- `Flash Size = 4MB`

### 第三步：如果串口监视器开着，先关掉
如果 `Serial Monitor` 打开着，先关掉，避免占用端口。

### 第四步：点 Upload
这一次上传，才是对 C3 的真正烧录验证。

## 如何判断结果

### 如果成功
如果成功上传，再打开 `Serial Monitor`，设置波特率：
- `115200`

正常应看到反复输出：
- `ESP32-C3 upload test start`
- `tick`
- `tick`
- `tick`

这就说明：
- Mac 端 Arduino IDE 正常；
- USB 烧录链路正常；
- 原来的 S3 板子更像是单板问题。

### 如果失败
如果这次新的最小程序上传时，报错目标端口已经变成：
- `/dev/cu.usbmodem101`

那我们再根据 **新报错** 继续判断。
