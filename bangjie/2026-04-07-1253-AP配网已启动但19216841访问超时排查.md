## 本轮最关键判断

当前最可能不是代码问题，而是下面几类客户端侧问题：

### 原因 A：Mac 没有真正连接到 `InkTime-090`
这是当前第一怀疑项。

因为：
- ESP32 已经明确打印出 AP 启动成功；
- 如果 Mac 仍停留在原来的公司/家庭 Wi‑Fi，浏览器访问 `192.168.4.1` 往往就会超时。

### 原因 B：Mac 连上了热点，但走了代理 / VPN / 网络扩展
如果浏览器或系统启用了：
- 代理
- 科学上网工具
- 某些网络过滤插件

则本地 `192.168.4.1` 请求可能被错误转发或拦截。

### 原因 C：浏览器没有真的走 HTTP 明文访问
项目当前是：
- `http://192.168.4.1/`

如果浏览器自动尝试 HTTPS、自动升级、或插件重写请求，也可能访问失败。

### 原因 D：Mac 已连接 AP，但系统网络没有切到该接口
这种概率较低，但偶尔会出现：
- Wi‑Fi 看似已连上热点；
- 实际默认路由仍不正确；
- 浏览器请求没发到 ESP32 这张网卡上。

## 官方资料佐证

根据 Espressif 的 Arduino-ESP32 Wi‑Fi API 文档：
- `WiFi.softAP(ssid, password)` 用于把 ESP32 配置为 AP；
- `softAPIP()` 用于获取 AP IPv4 地址；
- AP 模式下 ESP32 可以接收来自其他设备的连接，并可在其上运行 HTTP 服务。

参考：
- [Arduino ESP32 Wi-Fi API](https://docs.espressif.com/projects/arduino-esp32/en/latest/api/wifi.html)

这与当前截图中：
- `softAP result = OK`
- `AP IP = 192.168.4.1`
完全一致。
