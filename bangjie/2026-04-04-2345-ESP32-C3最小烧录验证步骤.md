# ESP32-C3最小烧录验证步骤

记录时间：2026-04-04 23:45（Asia/Shanghai）

## 本轮目标

本轮不是继续点亮 7.3 寸四色墨水屏，而是先用用户新提供的 `ESP32-C3` 圆板做一个 **最小烧录验证**，用来判断：
- Mac 端 Arduino IDE 是否工作正常；
- USB 线和串口链路是否工作正常；
- 之前那块 `ESP32-S3` 是否大概率为单板故障。

## 当前判断

用户这块新板核心模组为：
- `ESP32-C3-MINI-1U`

这块板子不适合直接继续当前墨水屏项目，但适合做“对照烧录测试板”。

## Arduino IDE 推荐设置（C3 最小测试）

对于这块板子，当前先用最保守设置：
- `Board = ESP32C3 Dev Module`
- `Upload Speed = 115200`
- `USB CDC On Boot = Enabled`
- `Flash Size = 4MB`（先按默认/保守值）
- 其他设置先保持默认

说明：
- 这一步的目标只是让板子成功烧录并打印串口日志；
- 不是为了跑墨水屏项目；
- 因此不要先纠结屏幕接线，也不要接墨水屏。

## 本轮实操步骤

### 第一步：先把 C3 板子单独接上 Mac
要求：
- 只接这块 `ESP32-C3` 圆板；
- 不接墨水屏；
- 不接其他外设；
- 只保留 `C3 裸板 + USB 线 + Mac`。

### 第二步：打开 Arduino IDE
在 Arduino IDE 里只做这几项：
- `Tools -> Board -> ESP32 Arduino -> ESP32C3 Dev Module`
- `Tools -> Port -> 选择新出现的串口`
- `Tools -> Upload Speed -> 115200`
- `Tools -> USB CDC On Boot -> Enabled`
- `Tools -> Flash Size -> 4MB`（如果菜单里能看到）

### 第三步：不要先开我们的 InkTime 工程
先不要打开：
- `esp32/ink-display-7C-photo/ink-display-7C-photo.ino`

因为当前第一目标不是适配项目，而是验证“C3 能不能正常烧进去一个最小程序”。

### 第四步：先烧一个最小串口测试程序
建议先新建一个最小测试草图，只做两件事：
- 启动串口；
- 每秒打印一行日志。

如果后续需要，我可以直接给用户一份最小测试代码。

## 如何判定结果

### 成功
如果出现下面结果，则说明：
- 上传成功；
- 串口监视器能看到周期性输出；
- Mac 端 Arduino IDE / USB 线 / 基础烧录环境是正常的；
- 之前那块 `ESP32-S3` 板子本身有较大概率存在硬件问题。

### 失败
如果这块 `C3` 也在最小程序上传时报类似错误，那么就说明：
- 问题不一定只在之前的 `S3` 板子；
- 还需要继续回头查 Mac 端串口环境、驱动兼容、USB 线材、板载 USB 桥等因素。

## 给用户的下一句执行话术

下一步直接让用户做：
- 把这块 `ESP32-C3` 单独插上 Mac；
- 在 Arduino IDE 里选 `ESP32C3 Dev Module`；
- 把新出现的端口告诉我；
- 然后再进行最小测试程序烧录。
