## 现在应该怎么继续

### 第 1 步：在 Arduino IDE 里选对端口
进入：
- `Tools -> Port -> /dev/cu.usbserial-110`

### 第 2 步：打开串口监视器
进入：
- 右上角 `Serial Monitor`

### 第 3 步：把波特率改成 115200
必须设置为：
- `115200`

原因：
- 之前已经出现过一次乱码截图，且当时串口监视器是 `9600 baud`；
- 主程序运行时大概率使用 `115200` 输出日志；
- 如果这里仍然设成 `9600`，日志会继续乱码，无法判断屏幕初始化走到哪一步。

### 第 4 步：按一下开发板上的 RST
在串口监视器开着的情况下：
- 轻按一下 `RST`

目的：
- 重新抓一份完整启动日志；
- 看程序是否再次成功启动；
- 看是否进入屏幕初始化和热点启动。

## 本轮重点要观察的日志内容

用户下一次反馈时，优先提供以下信息：

### 1. 是否出现启动头
例如类似：
- `ESP-ROM:esp32s3-20210327`

### 2. 是否再次出现热点配置日志
例如类似：
- `[CFG] softAP result = OK`
- `AP SSID = InkTime-xxxx`

### 3. 是否出现和墨水屏相关的异常
重点看是否有：
- 卡住不动；
- 一直 busy；
- reset 失败；
- SPI 初始化异常；
- 内存分配失败；
- 死循环重启。

## 如果串口正常但屏幕还是没反应
那下一步要回到硬件核线：
- `BUSY -> GPIO14`
- `RST/RES -> GPIO13`
- `DC -> GPIO12`
- `CS -> GPIO11`
- `SCLK/SCL -> GPIO10`
- `DIN/MOSI/SDA -> GPIO9`
- `VCC -> 3V3`
- `GND -> GND`

并特别提醒：
- 转接板上的 `SCL/SDA` 在这里不是 I2C，而是 SPI 对应脚位；
- 只要把 `SDA` 当成 I2C 数据脚去接错，屏幕就很可能完全不刷新。
