## 2. 结合你照片再次确认硬件

### 2.1 ESP32 板子识别

从 `/Users/apple/Downloads/墨水屏/esp32.jpeg` 可见：

- 模组丝印：`XH-S3E`
- 罩子丝印：`N16R8`
- 板上有 `RST`、`Boot` 按键
- 板载 USB-C
- 适合作为 Arduino IDE 中的 `ESP32S3 Dev Module` 来使用

`N16R8` 这点很关键，基本可按：

- `16MB Flash`
- `8MB PSRAM`

来理解。  
因此这块板子非常适合当前 InkTime 7.3 寸四色屏固件。

### 2.2 墨水屏转接板识别

从 `/Users/apple/Downloads/墨水屏/转换板.jpeg` 可见：

- 绿色小板丝印：`七色EPD转接板`
- 8Pin 接口丝印顺序：
  - `BUSY`
  - `RST`
  - `DC`
  - `CS`
  - `SCL`
  - `SDA`
  - `GND`
  - `3V3`

这里的 `SCL / SDA` **不是 I2C**，在这个项目里要按 SPI 理解：

- `SCL = SPI SCLK`
- `SDA = SPI MOSI / DIN`

### 2.3 最终接线表

当前仓库固件默认引脚是：

```cpp
#define PIN_EPD_BUSY 14
#define PIN_EPD_RST  13
#define PIN_EPD_DC   12
#define PIN_EPD_CS   11
#define PIN_EPD_SCLK 10
#define PIN_EPD_DIN  9
```

因此接线直接按下面连：

| 转接板丝印 | ESP32-S3 引脚 |
|---|---|
| BUSY | GPIO14 |
| RST | GPIO13 |
| DC | GPIO12 |
| CS | GPIO11 |
| SCL | GPIO10 |
| SDA | GPIO9 |
| GND | GND |
| 3V3 | 3V3 |

**注意：只能接 3V3，不要接 5V。**

### 2.4 你新补充的教程截图，和当前结论是什么关系

你补充的那张教程图，核心内容是：

- 环境：`Arduino 2.X + ESP32 (2.0.18) + GxEPD2`
- 驱动示例：`GxEPD2_7C<GxEPD2_730c_GDEY073D46, GxEPD2_730c_GDEY073D46::HEIGHT / 4>`
- 说明适用于：`7.3 七色` 和 `7.3 四色`
- 示例 IO：`CS=33 / DC=27 / RST=26 / BUSY=25`
- 示例 SPI：`SPI.begin(13,12,14,15)`

这份资料和前面的分析 **是相互印证，不是互相推翻**：

1. 它再次证明 `GxEPD2_730c_GDEY073D46` 这个驱动入口是对的。
2. 它说明 7.3 寸彩色屏确实常见于“普通 ESP32 开发板 + GxEPD2 + 自定义 SPI”的玩法。
3. 但它给的是“另一套开发板示例引脚”，**不是 InkTime 这份 S3 固件当前使用的 GPIO**。

所以对你现在这块 `ESP32-S3 XH-S3E / N16R8` 来说，依旧建议：

- 不要先照截图去改成 `25/26/27/33`
- 先沿用 InkTime 固件已经写好的 `GPIO9~GPIO14`

## 5. 现在最接近可跑通的 Arduino IDE 设置

等你把板子接到这台电脑，或者你在自己的电脑上操作 Arduino IDE 时，建议直接按下面选：

- Board：`ESP32S3 Dev Module`
- USB CDC On Boot：建议 `Enabled`
- Flash Size：优先 `16MB`
- PSRAM：`OPI PSRAM` 或 `Enabled`
- Partition Scheme：默认先用即可
- Port：选择实际出现的 `usbmodem / usbserial`

**PSRAM 一定要开。**  
因为这份 7.3 寸四色屏固件会优先申请大约 `384000` 字节 framebuffer，没开 PSRAM 虽然不一定 100% 立刻崩，但运行裕量会明显偏紧。

这对当前排障很有价值，因为它提供了一组偏保守、偏成熟的兼容性基线。

## 6. 烧录前还必须再核对的一项

固件里现在还是：

```cpp
#define DAILY_PHOTO_PATH_PREFIX "/static/inktime/yourdownloadkey/photo_"
```

而服务端 `config.py` 里需要对应：

```python
DOWNLOAD_KEY = "yourdownloadkey"
```

如果这两个值不一致，会出现：

- ESP32 能联网
- 但下载图片返回 `404`
- 屏幕不刷新或只显示失败状态

也就是说，**WiFi 连上了不代表整条链路通了，下载 key 一致性必须检查。**
