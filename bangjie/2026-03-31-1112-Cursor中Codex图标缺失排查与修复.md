# Cursor中Codex图标缺失排查与修复

## 现象

- 在 Cursor 左侧 Activity Bar 看不到 Codex 图标。
- 但在命令面板中可以搜索到 `View: Show Codex`。

## 结论

这不是 Codex 扩展完全丢失，而是当前本机 Cursor 与已安装的 `openai.chatgpt` 扩展之间存在视图容器兼容问题。

扩展的清单声明里，Codex 会优先注册到 `secondarySidebar`：

- `codexSecondaryViewContainer`
- `chatgpt.sidebarSecondaryView`

但本机 Cursor 日志显示该容器不存在，因此 Codex 入口没有正常出现在左侧 Activity Bar。

## 本机证据

### 1. 扩展本身已安装

本机存在多个 Codex 扩展版本：

- `/Users/apple/.cursor/extensions/openai.chatgpt-26.318.11754-darwin-arm64`
- `/Users/apple/.cursor/extensions/openai.chatgpt-26.323.20928-darwin-arm64`
- `/Users/apple/.cursor/extensions/openai.chatgpt-26.324.21329-darwin-arm64`
- `/Users/apple/.cursor/extensions/openai.chatgpt-26.325.21211-darwin-arm64`
- `/Users/apple/.cursor/extensions/openai.chatgpt-26.325.31654-darwin-arm64`

### 2. 日志显示 secondary sidebar 容器不存在

在以下日志中可以看到明确警告：

- `/Users/apple/Library/Application Support/Cursor/logs/20260325T111647/window14/renderer.log`

关键内容：

- `Extension 'openai.chatgpt' wants API proposal 'chatSessionsProvider' but that proposal DOES NOT EXIST.`
- `Extension 'openai.chatgpt' wants API proposal 'languageModelProxy' but that proposal DOES NOT EXIST.`
- `[openai.chatgpt]: View container 'codexSecondaryViewContainer' does not exist and all views registered to it will be added to 'Explorer'.`

这说明：

- 扩展被激活了，所以命令面板还能搜到 Codex。
- 但扩展注册的侧栏容器与当前 Cursor 不完全兼容，导致图标入口缺失或位置异常。

## 已执行修复

已直接修改本机已安装的 Codex 扩展 `package.json`，将 Codex 视图从“依赖 `secondarySidebar`”改为“固定注册到 `activitybar`”。

修改后的效果：

- `viewsContainers` 只保留 `activitybar`
- `views` 只保留 `codexViewContainer`
- 去掉了依赖 `chatgpt.doesNotSupportSecondarySidebar` 的 `when` 条件

已修改文件：

- `/Users/apple/.cursor/extensions/openai.chatgpt-26.318.11754-darwin-arm64/package.json`
- `/Users/apple/.cursor/extensions/openai.chatgpt-26.323.20928-darwin-arm64/package.json`
- `/Users/apple/.cursor/extensions/openai.chatgpt-26.324.21329-darwin-arm64/package.json`
- `/Users/apple/.cursor/extensions/openai.chatgpt-26.325.21211-darwin-arm64/package.json`
- `/Users/apple/.cursor/extensions/openai.chatgpt-26.325.31654-darwin-arm64/package.json`

并已用 `jq` 验证这些 JSON 仍然合法。

## 你现在需要做的事

### 方案一：优先执行

在 Cursor 中执行：

- `Developer: Reload Window`

如果还没恢复，再完全退出 Cursor 后重新打开。

### 方案二：如果图标仍未出现

继续检查下面两项：

- 右键左侧 Activity Bar，确认 `Codex` 没有被手动隐藏。
- 执行 `View: Reset View Locations`，把被挪动的视图恢复到默认位置。

## 风险说明

- 这次修复属于本机扩展清单热修复。
- 如果 Cursor 后续自动更新 `openai.chatgpt` 扩展，修改可能被覆盖。
- 若再次消失，可重复同样处理，或者等 Cursor / Codex 扩展官方完成兼容修复。

## 简短判断

本次问题的核心原因不是“没安装 Codex”，而是“Codex 扩展把入口挂到了 Cursor 当前并未正确承载的 `secondarySidebar` 容器上”，所以出现“能搜到命令，但看不到图标”的现象。
