## 当前应使用的配网参数

如果你现在使用的是手机热点 `mengbj_iPhone`，那么 ESP32 首次配网 / 修改配置时应填：

### Wi-Fi

- SSID：`mengbj_iPhone`
- Password：填你的手机热点密码

### Server

- Host:Port：`172.20.10.9:8765`

### 特别注意

不能填：

- `127.0.0.1:8765`

原因：

- `127.0.0.1` 对 ESP32 来说代表 **它自己**，不是你的 Mac
- 所以 ESP32 访问 `127.0.0.1:8765` 永远访问不到 Mac 上的 Flask 服务

### 第 5 步：观察墨水屏有没有刷新动作

这时候盯着墨水屏看，正常现象通常包括：

- 屏幕先闪一下

- 黑 / 红 / 黄 / 白测试图案出现
- 刷新时间较长，可能要几十秒

对于四色墨水屏：

- 刷新通常比黑白屏慢
- 中间出现闪屏、花屏、局部过渡，都不一定是异常
- 关键是最后能不能稳定显示测试图案

## 如果串口已经 200，但屏幕还是没反应，重点排查哪里

如果已经确认：

- Wi-Fi 连上了
- HTTP 返回 `200`
- `Content-Length=384000`
- 数据也读满了

但屏幕完全不刷新，那么下一阶段就不是网络问题，而是**硬件接线 / 排线 / 屏驱动链路问题**。

优先检查：

- 转接板供电是否稳定：`3V3` / `GND`
- SPI 线是否对应：
  - `DIN/MOSI -> GPIO9`
  - `CLK -> GPIO10`
  - `CS -> GPIO11`
  - `DC -> GPIO12`
  - `RST -> GPIO13`
  - `BUSY -> GPIO14`
- 排线方向是否插反
- FPC 是否压紧
- 墨水屏转接板和主控板之间公共地是否真实接通
- 杜邦线是否接触不良

### 当前代码使用的引脚定义

以工程 `/Users/apple/bangjie/dev_jy/InkTime/esp32/ink-display-7C-photo/ink-display-7C-photo.ino` 为准：

- `EPD_CS = 11`
- `EPD_DC = 12`
- `EPD_RST = 13`
- `EPD_BUSY = 14`
- `EPD_SCK = 10`
- `EPD_MOSI = 9`
- `FACTORY_RESET_PIN = 38`

所以接线必须和这里一致，不能按别的 Waveshare 示例图随便套。
